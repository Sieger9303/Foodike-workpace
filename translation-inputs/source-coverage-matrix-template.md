# Source Coverage Matrix Template

## Purpose

This deterministic template enumerates source feature candidates from source files, key methods/functions, resources, ids, dependencies, and risk tags. The agent-maintained `source-coverage-matrix.json` should track the real migration status for these features, including sourceBehavior, targetBehavior, parityLevel, targetEndpoints, tests, and verificationEvidence for closed critical/high items.

## Status Values

`deferred`, `implemented`, `implemented-simplified`, `not-applicable`, `not-covered`, `planned`, `platform-replaced`, `unreviewed`

## Feature Id Policy

- Feature id version: `2`
- Ids include a short hash of the full source path, feature index, feature name, and feature point.

## Deferred Required Fields

`sourceEvidence`, `deferredReason`, `targetFallback`, `risk`, `futureCompletionPath`
